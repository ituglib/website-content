Instructions for use of OpenSSL 1.0.2

Use the OSS command
  tar xzf openssl-package.tar.gz -C / -p --no-same-user
to install OpenSSL. You must install the package in the exact same location as in the tar file. Add the /usr/local/lib to your _RLD_LIB_PATH variable when OpenSSL 1.0.2 is needed. Other versions of OpenSSL should be installed elsewhere.

Due to new features added in the latest OSS System Libraries, binaries built with those will not function correctly on older RVUs. The following table lists the minimum RVU for each platform that can correctly run git version 1.0.2u, and above:

Platform Minimum RVU
TNS/E J06.22
TNS/X L19.08
TNS/V L19.08

Note: The OpenSSL team has deprecated version 1.0.2u as of December 2019. Security patches are no longer being created for that version.

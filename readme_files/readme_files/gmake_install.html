Instructions for use of GMAKE 4.1g and above:

Upload the gmake-package.pak.bin file to GUARDIAN in binary. The file can be named anything in GUARDIAN, typically gmakepak.

Change the file code of the binary file to 1729 as follows:
  FUP ALTER gmakepak,CODE 1729

Run the UNPAK utility to install the GMAKE object in your desired location:
  UNPAK gmakepak,*.*.*,listall,myid,vol $myvol.subvol,tapedate

Secure the GMAKE object based on your system's needs:
  FUP SECURE gmake,"NCNC"

The files in the PAK archive default to being owned by SUPER.SUPER and secured "NUNU".

Due to new features added in the latest Native Libraries, binaries built with those may not function correctly on older RVUs. The following table lists the minimum RVU for each platform that can correctly run gmake version 4.1g, and above:

Platform Minimum RVU
TNS/E J06.22
TNS/X L19.08
TNS/V L19.08

If you require new features or bug fixes and your system is older than the above RVUs, you may have to download the source package from GitHub and build the new version yourself. This requires experience with building open source tools on OSS.

The source repository for GMake currently resides at: https://github.com/ituglib/gmake. If you find any issues, want new features, and wish to contribute, you can do so using the GitHub site.

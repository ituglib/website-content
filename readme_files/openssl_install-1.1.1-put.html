Instructions for use of OpenSSL 1.1.1 (PUT Library)

Use the OSS command
  tar xzf openssl-package.tar.gz -C / -p --no-same-user
to install OpenSSL. You must install the package in the exact same location as in the tar file. Add the /usr/local-ssl1.1_put/lib to your _RLD_LIB_PATH variable when OpenSSL 1.1.1 PUT is needed. Other versions of OpenSSL should be installed elsewhere.

Due to new features added in the latest OSS System Libraries, binaries built with those will not function correctly on older RVUs. The following table lists the minimum RVU for each platform that can correctly run git version 1.1.1g, and above:

Platform Minimum RVU
TNS/E J06.22
TNS/X L19.08
TNS/V L19.08

Caution: Do not mix versions of OpenSSL. There is no binary compatibility between version 1.0.2 and 1.1.1 of OpenSSL. The first directory encountered in _RLD_LIB_PATH is selected by the OSS loader. Do not use a threaded library with an unthreaded program and vice versa.

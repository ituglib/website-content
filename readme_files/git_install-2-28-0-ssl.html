Instructions for use of git versions 2.28.0 and above with SSL 1.1+

Use the OSS command
  tar xzf git-package.tar.gz -C / -p --no-same-user
to install git. Make sure to have the directory containing the OpenSSL 1.1.x version of libcrypt.so in your _RLD_LIB_PATH variable when git is used.

Note: This package installs in a fixed path to ensure separation from incompatible SSL instances.

Due to new features added in the latest OSS System Libraries, binaries built with those will not function correctly on older RVUs. The following table lists the minimum RVU for each platform that can correctly run git version 2.28.0, and above:

Platform Minimum RVU
TNS/E J06.22
TNS/X L19.08
TNS/V L19.08

If you require new features or bug fixes that were released after 2.16.2.9, you will have to download the source package and build the new version yourself. This requires experience with FLOSS and with building open source tools on OSS.

If you don't need these new features you should continue to use version 2.16.2.9 instead of the latest version.
